  1. Join

Node.js

  1. 개발 환경 설정
  • LTS = long term soft 장기적인 업데이트와 지원이 가능 실제 서비스 환경에서 사용해야할 버젼
  • Node —v, npm—v

  1. I/O 와 프로그래밍 패러다임
  • 먼져 Node 가 생겨난 이유는 기존에 web server(apache…)는 쓰레드와 같은 동시적인 프로그래밍이 불가능 했다.
  • 속도와 관련된 성능이 큰 비중을 차지하고 관련) 네트워크 트래픽을 통해서 라우팅(전달)이 중요

  • Non-blocking vs blocking 는 프로세스 진행중 다른 프로세스를 block해서 성능 저하가 될수 있다. 따라 node 는 non-blocking 을 사용
  • js는 이벤트 주도 방식의 언어 반대는 쓰레드풀 방식
