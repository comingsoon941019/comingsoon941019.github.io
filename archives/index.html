  • 요금 명세 : https://aws.amazon.com/ko/pricing/?nc2=h_ql_pr_ln

    • 요금을 내는 방식이 총 4가지(온디맨드, 스팻인스턴스, 예약인스턴스, 전용호스팅)

  • 프리티어 확인후 사용이 필요

  1. EC2 사용

  • AWS 서비스 콘솔에서 사용

  • 컴퓨터 한대를 사용할 때 인스턴스 하나를 시작한다고한다.

  • 인스턴스 시작 버튼을 클릭하면 프리티어로 인스턴스 가능한 목록을 확인 가능하다.

  • ubuntu 18.04 LTS 프리티어를 인스턴스 하고 시작

  • 키패어 생성 화면을 확인 할 수 있다. 새 키페어를 생성한후 키페어 이름설정 키페어 다운로드 하여 키를 관리한다. 유출, 누실을 주의하자.

  • pending을 거쳐 인스턴스 상태가 running이 되면 이제 원격으로 컴퓨터를 사용할 준비가 끝났다.

  • EC2를 사용하기 위해 mac을 쓰고있는 나는 Cyberduck을 설치

  • sftp 모드를 설정한 뒤 퍼블릭 DNS를 주소로 인스턴스와 연결한다.

  • ssh -i .pem ubuntu@.compute.amazonaws.com 다음과같이 사용된 DNS 에서 명령어를 사용하면 인스턴스한 우분투의 터미널로 접속이 가능하다.

  • MySQL 설치 전에 소스 디렉토리 내에 build를 위한 디렉토리를 추가 생성하고 그 하단에서 작업 진행을 해야 한다.

groupadd mysql 은 mysql이라는 그룹을 만들어서 관리하겠다는 뜻
useradd는 새로운 사용자를 추가하겠다는 뜻 -r 옵션은 system account를 만든다.
-g 옵션은 어떤 그룹에 넣고 관리할 것인지, -s 옵션은 유저의 로그인 shell 을 지정
cat /etc/passwd 를 통해 유저 목록을 확인할 수 있다.

  • /usr/local/mysql 이동후 mysql-files 디렉토리 생성

  • 디렉토리 사용자 및 그룹 권한 설정

  • 기본 데이터베이스 생성.

    이때 데이터베이스를 초기화 했을때 문구가 출력되는 데 맨아래에 temporary password가 나온다 후에 mysql 서버 연결에 필요하니 메모

Vim install, setting

  • 우분투를 사용할 때 vi 편집기를 사용할 때 vim설치를 해주지 않고 사용하면 매우매우 불편하다. 먼저 설치를 하고 시작하자.

  • apt 업데이트 이후 설치

  • vim 설치가 끝이 아니라 설정까지 해줘야 vim사용이 가능하다.

  • vim설정 파일 진입

Apache 2.4.43 compile install

  • 본격적으로 apache 수동 설치시작한.

  • /usr/local/ 경로에 설치하는 것이 바람직

  • /usr/local/src 소스파일 저장

  • 설치 하는 순서는 보통 wget-> xvfz tar -> ./configure -> make -> make install

  • 소스컴파일에 apr, apr-util, pcre 와 같은 의존성 패키지는 apt로 설치

  • apr : 아파치 HTTP 서버 2.x.의 핵심이며 휴대용 라이브러리, 이런 APR은 고급 IO 기능에 대한 접근을 포함하여 OS 수준의 기능, 기본프로세스 처리 등 많은 용도로 사용된다.

  • PCRE : 펄 호환 정규 표현식으로 정규식 패턴 일치를 구현하는 함수의 집합 아파치 2.4버전 설치시 필요한 패키지

  1. 소스를 저장할 디렉토리 생성
  • 다음 명령어를 수행하는데 어려움이 있다면 리눅스의 디렉토리 구조를 한번 보고 오는게 좋다. https://webdir.tistory.com/101
  1. apr,apr-util 다운

  • wget : web get 의 약어로 웹상의 파일을 다운받을 때 사용

  • tar xvfz : tar.gz 형식으로 압축되어있는 파일을 압축 해제하는 명령어

  1. apr 설치

  2. apr-util 설치

  • 여기서 많이 삽질했는데 apr-util 설치전 필수적으로 libexpat1-dev 라는 패키지를 먼저 설치 해줘야 한다. 이 패키지가 설치되지 않은 상태로 설치되면 제대로 설치가 되지 않아 나중에 apache 설치시 오류가 나서 재설치하는 번거러움이 생긴다.
  • 다음으로 apr-uilt 설치

  1. PCRE 설치

  2. apache 2.4.43 설치

  • apache 공식 사이트 소스 설치 : http://httpd.apache.org/docs/current/install.html

  • apache 최신 버젼 링크 : http://httpd.apache.org/download.cgi

  • 다운로드시 최신 버젼을 확인하여 설치한다.

  • 다운로드후 configure, make, make install 순서로 설치

  • configure 스크립트를 사용해 아파치 소스트리 구성, 자세한 script 구성은 http://httpd.apache.org/docs/current/en/programs/configure.html

  • 설치가 완료 되었으면 실행을 해본다.

  • 다음과 같은 오류가 발생하였는데 일단 동작은 하지만 찾아봐야할거 같다.. AH00558: httpd: Could not reliably determine the server’s fully qualified domain name, using 127.0.1.1. Set the ‘ServerName’ directive globally to suppress this message

  • netstat와 curl 이 동작하지 않는다면 설치

  1. httpd 서비스 등록

가상환경에 우분투 설치

Virtual Box

  • mac을 사용하고 있지만 ubuntu 공부를 위해 가상환경에서 apm 설치를 하고자 한다.

  • 가상환경은 무료로 사용가능한 Virtual Box를 사용한다.

Ubuntu 20.04 LTS

  • 우분투 (ubuntu)는 데비안 GNU/리눅스 기반으로 개발된 배포판

  • LTS(장기 지원판)이며 2년에 한번씩 배포됨

  • 해당 사이트에서 이미지 파일을 받아 가상머신에 설치 할 예정 다운받는 속도는 매우 느리다.

virtual box 에 ubuntu 설치

  • virtual box에 새로만들기 클릭후 가상머신의 이름과 버젼, 종류를 선택

  • 이후 메모리 크기를 설정 하는데 공홈에서 4GB를 권장하지만 여유가 된다면 이상을 추천한다.

  • 하드 디스크 지정 후 VDI 선택

  • 하드 디스크를 동적 할당 또는 고정크기로 정하는데 호스트 컴퓨터 하드디스크에 여유가 없음 동적 할당을 추천

  • 다음으로 하드의 크기를 정할때가 문제 생각 이상으로 apm 설치가 용량이 많이 들어 최소 40기가 정도로 설정하는게 번거롭게 재설치를 하지 않아도 될듯하다.

  • 다음으로 아까 받았던 ubuntu img를 올려 설치를 시작한다.

  • 이제 설치를 하다 보면 우분투 해상도 문제로 버튼을 클릭하지 못하는 문제가 발생하는데 이때 우분투를 종료후 virtualbox의 디스플레이 설정에서 그래픽컨트롤러를 VBoxSVGA로 변경
