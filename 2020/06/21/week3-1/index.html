server_week_3-1 ( DataGrip, MySQL, Query )

Jun 21, 2020 in raising project

Intro

서버 3주차

DataGrip과 MySQL로 온라인 커머스 DB를 설계해 보고자 한다.

DB 설계

  1. 요구 사항 수집 분석
  • 먼저 실제 세계 에서 어떤 시스템이 필요한지 요구사항 정리
  • 요구사항에 대해 필요한 데이터 분석
  1. 설계방법
  • E-R 모델과 릴레이션 변환 규칙을 이용한 설계
  • 정규화를 이용한 설계

설계하고자 하는 기능을 문장으로 표현하는게 효과적이다.

  • 다음과 같이 DB 설계에 공부가 미흡한상태로 바로 설계 부터 했는데 쿼리문 작성시 구조가 잘못되어 불편함이 많았다.
  • 여기서 테이블은 정말 쉽게 구조화 하고 쿼리문까지 생성이 가능한 https://aquerytool.com/ 사이트를 이용한다.

Query

  1. Join

  • 다음 그림에서 join의 개념을 한눈에 볼 수 있다.

  • join의 사용 쿼리는

  • Left Join : A,B 두개의 테이블 중 A값의 전체와, A의 KEY 값과 B KEY 값이 같은 결과를 리턴

  • Inner Join : ID값이 서로 중복되는 값만 나타낸다.

  • Full Outer Join : MySQL은 FULL OUTER JOIN이 없다. 그래서 LEFT JOIN 과 RIGHT JOIN을 이용해 FULL OUTER JOIN을 사용
