모의 외주 대비 Node js 공부

Node.js

  1. 개발 환경 설정
  • LTS = long term soft 장기적인 업데이트와 지원이 가능 실제 서비스 환경에서 사용해야할 버젼
  • Node —v, npm—v
  • Vscode 에서 개발 하며 standard js패키지를 추가 설치하여 js 환경을 좀더 간편하게 개발할 수 있게 한다.
  1. I/O 와 프로그래밍 패러다임
  • 먼져 Node 가 생겨난 이유는 기존에 web server(apache…)는 쓰레드와 같은 동시적인 프로그래밍이 불가능 했다.
  • 속도와 관련된 성능이 큰 비중을 차지하고 관련) 네트워크 트래픽을 통해서 라우팅(전달)이 중요
  • 비동기 vs 동기 위와 같은 문제를 해결하기 위한 node은 2가지 모델
  • 동기화는 프로세스를 진행중 끝날때 까지 다른 프로세스는 대기 하여 같은 시기에 (동기화) 하게 된다
  • 비동기는 서로 다른 프로세스는 다른 시점에서 처리 한다. 이를 위해 node js 는 이벤트 루프라는 방법을 사용, 정확히는 node js 는 내부에 v8이라는 js의 무언가를 사용
  • Non-blocking vs blocking 는 프로세스 진행중 다른 프로세스를 block해서 성능 저하가 될수 있다. 따라 node 는 non-blocking 을 사용
  • js는 이벤트 주도 방식의 언어 반대는 쓰레드풀 방식

JavaScript

  1. JavaScript
  • 스크립트 언어
  • 인터프리터 언어 : 컴파일을 거치지 않고 바로 실행하는 인터프리터 언어
  • 자바랑은 이름만 비슷하며 전혀 다른 언어
  • 웹의 컨텐츠를 동적으로 변경할때 사용하는 언어
  • Js 는 electron이라는 데스크탑 어플리케이션도 가능하다
  1. Js Expression
  • 값을 만들어내는 간단한 코드를 표현식이라고 함 ex)1,”a”,true 등등
  • alert();
  1. Keywords
  • Var // 변수를 선언하기 위한 예약어
  1. Mothereif
  • js에서 유효한 변수명을 확인해주는 사이트
  1. 주석
  • //, /* */
  1. 변수
  • Let a = value; // 변수를 선언
  • Let? Var?
  1. Var / let, const
  • Es 5 = var
  • Es 6 = let, const
  1. Var 와 Hoisting
  • 함수를 밑에 선언하고 위에서 호출해서 사용이 가능하다, 변수도 가능하다 이를 Hoisting이라고 한다.
  • Hoisting은 선언부가 밑에 있을때 선언부만 위로 끌어올려서 사용하는 식이다.
  • 하지만 ES6의 let 으로 선언 했을 경우에는 hoisting 현상이 발생하지 않는다.
  1. 자료형
  • 기본적으로 js의 자료형은 동적 타이핑이라고 한다.
  • null, undefined : 비어있는 값을 null, 값을 지정해 주지 않았을 경우 undefined
  • Int형과 같이 Number라는 변수가 있다.
  • Template string = ${name} : ss
  • Symbol : 은 같은 값이어도 다르게 인식 고유한 값을 만들어 낼때 사용
  1. Class
  • es6부터 지원
  • 클래스는 hoisting이 발생하지 않는다.
  • Constructor : class의 생성자
  • getter와 setter를 지원한다.
  • class child extends parents 상속
  1. Promise
  • Es6 부터 지원
  • 생성자를 통해서 promise 객체를 만들 수 있다.
  • 생성자의 인자로 excutor라는 인자를 사용
  • Promise 관련 문법은 따로 포스팅
  1. Arrow function
  • 이것또한 Es6부터 지원
